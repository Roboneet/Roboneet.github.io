An open source journey

Ever since I got selected to GSoC’18 program, I’ve been explaining what open source is to how amazing julia is to almost everyone I meet. Here’s an intro into how I got into open source and a small summary of my gsoc project that was done during the summer of 2018.

Open what?

“Open-source software (OSS) is a type of computer software whose source code is released under a license in which the copyright holder grants users the rights to study, change, and distribute the software to anyone and for any purpose” - Wikipedia

If you’ve ever tried downloading youtube videos, you’d probably know this cool software developed by thousands of contributors (If not, here it is...). Simply stated, all of the software for which it’s code is available for free* is an open-source software. People collaborate on creating open source softwares through websites like github.

Getting into open source is a good way to learn things. I started contributing to open source during the past summers while I was just learning how to use the Django framework. All the work done by student teams on BITS campus (including the fest websites, quizzing software code etc) are all made open and we collaborate using Github. As a part of all the little bit mentoring that I’ve done (on and off campus), I’ve cumulated a small list of stuff that you need to know to start contributing here

and GSoC?

To all the people who have no idea what my GSoC project is:

Over the summer, I worked on taking deep learning models made using Flux.jl to the web, specifically to http://fluxml.ai. This process includes making models using Flux, training them, creating an interface, and converting the models into javascript using FluxJS.jl. Also, got a few features added to FluxJS.jl including support for various layers and functions.
